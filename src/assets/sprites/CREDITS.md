# Overworld Character Sprite Credits

Each `<kind>.png` is a 128x128 sprite sheet laid out as 4 rows
(down, up, left, right) x 4 columns (idle, walk1, walk2, walk3) of 32x32
cells, no padding between cells. Each sprite's main mass (robe / body / wings)
is grayscale while accents (skin, eyes, mane, staff, hooves, teeth, fins) use
saturated intrinsic colors. The runtime tint in `src/ui/overworld/Sprite.jsx`
recolors only the low-saturation mass pixels, so one sheet per kind serves any
palette variant (gold / white / blue / black / red / green) while still looking
multi-colored.

## Provenance note

The task that introduced these sheets called for CC0 pixel-art sourced from
OpenGameArt / Kenney / itch.io. In the execution environment those hosts were
unreachable (HTTP 403 via both direct fetch and the agent fetch tool); only
GitHub and the pip/npm registries were on the network allowlist, and none of
them carried verifiable-CC0 sheets in the required 6-creature x 4-direction x
4-frame format. Rather than ship unverifiable third-party art, the sheets here
were generated programmatically (Pillow, supersampled + LANCZOS downscale for a
soft, anti-aliased, outline-free look matching `forest_tileset.png`). The
generator is committed at `tools/gen-sprites.py` and is fully deterministic:

```
python3 tools/gen-sprites.py
```

This original generated art is released into the public domain under
[CC0 1.0](https://creativecommons.org/publicdomain/zero/1.0/). No attribution
is required; it is recorded here for the project's own reference. If genuine
sourced CC0 art is preferred later, replace these PNGs (keeping the exact grid)
and update this file -- no code change is needed.
